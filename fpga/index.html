    FPGA & Hardware Projects

    A deeper look at my digital design and hardware engineering work — from pipelined CPUs and real-time image processing accelerators to FPGA-based music synthesis and LLM-driven hardware evaluation.

    Pipelined MIPS CPU

    EE 180 — Lab 3

    A five-stage pipelined MIPS processor implemented in Verilog and synthesized for FPGA. The pipeline includes full data-forwarding paths and hazard-detection logic with stalling to resolve data and control hazards. The design also supports atomic read-modify-write instructions for basic synchronization primitives.

    • Classic IF → ID → EX → MEM → WB pipeline stages
    • Forwarding unit eliminates most data-hazard stalls
    • Hazard detection with pipeline stalling for load-use dependencies
    • Atomic instruction support (LL/SC style synchronization)
    • Verified against a comprehensive hand-written assembly test suite
    • Synthesized and tested on FPGA development boards
    Verilog MIPS ISA FPGA Synthesis Pipelining Hazard Detection

    Sobel Edge Detection — Hardware Accelerator

    EE 180 — Lab 2

    A custom hardware accelerator for Sobel edge-detection filtering, designed for real-time video processing on FPGA. Uses line-buffered streaming pipelines to process incoming pixel data with minimal latency, avoiding the need to store full frames in external memory.

    • Line-buffer architecture for single-pass 3×3 convolution
    • Pipelined datapath for sustained one-pixel-per-clock throughput
    • Separate horizontal and vertical gradient computation with magnitude approximation
    • Real-time edge detection demonstrated on live video input
    • Validated with synthesis timing reports and on-board functional tests

    The same lab also included ARM NEON optimized software filters — single- and multi-threaded grayscale and Sobel implementations using NEON intrinsics, benchmarked for throughput and latency scaling across cores.

    Verilog FPGA Line Buffers Image Processing ARM NEON Multithreading

    Dynamic Music Player — FPGA

    EE 108 — Final Project

    A fully hardware-based music synthesizer and player implemented on FPGA. Custom Verilog modules generate chords, harmonics, echo, and dynamics in a pipelined waveform-generation pipeline. Notes are rendered on-screen through a VGA display controller.

    • Pipelined waveform generation with additive harmonic synthesis
    • Chord playback, echo/reverb effects, and dynamic volume control
    • Real-time VGA note visualization and display rendering
    • Modular design with reusable tone-generator and mixer components
    Verilog FPGA Audio Synthesis VGA Display Pipelining

    Programmable Bike Light

    EE 108 — Lab 2

    A configurable LED blinker system built from custom sequential logic in Verilog. A Beat32 timing base drives a hierarchical timer and shifter controlled by a master FSM. The blinker pattern, speed, and direction are all programmable via on-board inputs.

    • Beat32 clock divider as the timing base for all sequential logic
    • Hierarchical timer and barrel-shifter modules for pattern generation
    • Master FSM orchestrating mode transitions and user inputs
    • Custom D flip-flop primitives (DFF, DFFAR, DFFARRE) for clean reset behavior
    • Programmable blink rate, direction, and pattern via switches and buttons
    Verilog FPGA FSM Design Sequential Logic D Flip-Flops

    Password Authentication System

    EE 108 — Lab 1

    A purely combinational hardware authentication module on FPGA. Incoming usernames are matched against a Content-Addressable Memory (CAM) for constant-time parallel lookup, and passwords are validated through a custom hardware hash function with optimized propagation delay.

    • Content-Addressable Memory for simultaneous username comparison
    • Custom combinational hash function for password validation
    • Constant-time search — no sequential iteration through entries
    • Optimized critical path for minimal propagation delay
    • Parallel processing of username and hashed-password comparison
    Verilog FPGA Combinational Logic CAM Hashing

    LLM Hardware Eval Harness

    Research

    A research harness that evaluates how well large language models can generate correct hardware (Verilog) code. The harness includes two evaluation tasks: a MIPS32 CPU lab with a full Verilog testbench and hand-written assembly test suite, and a multiply-accumulate (MAC) task for quick sanity-checking of LLM code generation.

    • MIPS32 single-cycle and 5-stage pipelined CPU as the primary eval target
    • Hand-written assembly tests assembled with Clang/LLVM for big-endian MIPS
    • Automated candidate generation via OpenAI API with configurable model selection
    • Scoring pipeline: compile generated Verilog with Icarus Verilog, simulate against testbench, compute pass@k
    • MAC sub-task with prompt, testbench, and reference output for lightweight evaluation
    • Runs on macOS with Homebrew (Icarus Verilog + LLVM toolchain)
